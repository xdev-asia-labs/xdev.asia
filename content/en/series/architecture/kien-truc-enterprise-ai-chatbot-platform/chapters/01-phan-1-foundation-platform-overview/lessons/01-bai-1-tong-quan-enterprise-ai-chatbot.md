<h2 id="1-ai-chatbot-2026"><strong>1. AI Chatbot in 2026 — No longer Rule-Based</strong></h2>

<p>AI Chatbots have evolved since <strong>rule-based decision trees</strong> (if/else, keyword matching) to <strong>LLM-powered agents</strong> capable of reasoning, tool use, and self-regulation. Core differences:</p>

<table>
<thead>
<tr><th>Generation</th><th>Characteristics</th><th>For example</th><th>Limit</th></tr>
</thead>
<tbody>
<tr><td><strong>Gen 1: Rule-based</strong></td><td>Decision tree, keyword matching</td><td>Chatfuel, ManyChat</td><td>Only handle fixed scenarios</td></tr>
<tr><td><strong>Gen 2: NLU-based</strong></td><td>Intent classification + slot filling</td><td>Dialogflow, Rasa, Lex</td><td>Need large, brittle training data</td></tr>
<tr><td><strong>Gen 3: LLM-powered</strong></td><td>Foundation model + RAG</td><td>ChatGPT, Claude</td><td>Hallucination, no tool use</td></tr>
<tr><td><strong>Gen 4: Agentic AI</strong></td><td>Multi-agent, tool calling, planning, memory</td><td>OpenAI AgentKit, custom platforms</td><td>Complexity, cost, eval are difficult</td></tr>
</tbody>
</table>

<p>This series focuses on <strong>Gen 4 — Agentic AI Chatbot Platform</strong>, where the chatbot not only answers but also <strong>action</strong>: database lookup, ticket creation, payment processing, escalation to human agent.</p>

<h2 id="5-build-vs-buy"><strong>5. Build vs Buy Decision Framework</strong></h2>

<p>Framework for deciding when to build your own AI chatbot platform:</p>

<pre><code class="language-typescript">
// Build vs Buy Scoring Framework
interface DecisionCriteria {
  name: string;
  weight: number; // 1-5
  buildScore: number; // 1-10
  buyScore: number; // 1-10
}

const criteria: DecisionCriteria[] = [
  // Chọn BUILD khi:
  { name: 'Data Privacy (sensitive industry)', weight: 5, buildScore: 10, buyScore: 4 },
  { name: 'Custom domain logic', weight: 4, buildScore: 9, buyScore: 5 },
  { name: 'Integration depth', weight: 4, buildScore: 9, buyScore: 6 },
  { name: 'Multi-model flexibility', weight: 3, buildScore: 10, buyScore: 4 },
  { name: 'Long-term cost at scale', weight: 3, buildScore: 8, buyScore: 5 },

  // Chọn BUY khi:
  { name: 'Time to market', weight: 5, buyScore: 10, buildScore: 3 },
  { name: 'Team size < 5 engineers', weight: 4, buyScore: 9, buildScore: 4 },
  { name: 'Standard use case (FAQ/support)', weight: 3, buyScore: 9, buildScore: 5 },
  { name: 'Maintenance burden', weight: 3, buyScore: 8, buildScore: 4 },
];

function calculateScore(criteria: DecisionCriteria[]): { build: number; buy: number } {
  const build = criteria.reduce((sum, c) => sum + c.weight * c.buildScore, 0);
  const buy = criteria.reduce((sum, c) => sum + c.weight * c.buyScore, 0);
  const totalWeight = criteria.reduce((sum, c) => sum + c.weight, 0);
  return {
    build: Math.round((build / (totalWeight * 10)) * 100),
    buy: Math.round((buy / (totalWeight * 10)) * 100),
  };
}
</code></pre>

<p><strong>Rule of thumb:</strong></p>
<ul>
<li>Under 1,000 conversations/day + standard use case → <strong>Buy</strong> (Intercom, Zendesk)</li>
<li>10,000+ conversations/day + domain-specific + sensitive data → <strong>Build</strong></li>
<li>Hybrid: Buy SaaS for quick win, build custom for core differentiator</li>
</ul>
