<h2 id="1-vibe-coding-la-gi"><strong>1. What is Vibe Coding?</strong></h2>

<p><strong>Vibe Coding</strong> (or Vibecoding) is an AI-powered software development method in which programmers describe requirements in natural language and let the Large Language Model (LLM) automatically generate source code.</p>

<p>This term was coined by <strong>Andrej Karpathy</strong> — AI scientist, co-founder of OpenAI and former AI Director at Tesla — in February 2025:</p>

<blockquote>
<p>"There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists. It's possible because the LLMs are getting too good."</p>
<p>— Andrej Karpathy, February 2, 2025</p>
</blockquote>

<p>The term quickly became so popular that:</p>
<ul>
<li><strong>Merriam-Webster</strong> listed it as "slang & trending" in March 2025</li>
<li><strong>Collins English Dictionary</strong> choose it <strong>Word of the Year 2025</strong></li>
<li><strong>Y Combinator</strong> reported that 25% of startups in the Winter 2025 batch have a codebase that is 95% generated by AI</li>
</ul>

<h2 id="2-cach-vibe-coding-hoat-dong"><strong>2. How Vibe Coding works</strong></h2>

<p>Instead of the traditional process <em>think → write code → test → debug</em>, Vibe Coding according to the process:</p>

<pre><code>┌─────────────────────────────────────────────────┐
│            VIBE CODING WORKFLOW                  │
├─────────────────────────────────────────────────┤
│                                                  │
│  1. 💭 Describe (Mô tả yêu cầu bằng tiếng Việt/│
│        English cho AI)                           │
│           ↓                                      │
│  2. 🤖 Generate (AI sinh code tự động)           │
│           ↓                                      │
│  3. 👀 Review/Accept (Xem kết quả, accept/reject)│
│           ↓                                      │
│  4. 🔄 Iterate (Nếu chưa đúng, mô tả lại)      │
│           ↓                                      │
│  5. ✅ Ship (Deploy sản phẩm)                    │
│                                                  │
└─────────────────────────────────────────────────┘
</code></pre>

<p>Karpathy describes his experience:</p>
<ul>
<li>Talk to AI via voice (SuperWhisper)</li>
<li>Always press "Accept All" without reading the diff</li>
<li>Copy-paste error messages without any further comments</li>
<li>Code evolves beyond the developer's ability to understand</li>
</ul>

<p><strong>Simon Willison</strong> (famous developer) clearly distinguishes:</p>
<blockquote>
<p>"If an LLM writes every line of your code, but you've reviewed, tested, and understood it all, that's not vibe coding in my book — that's using an LLM as a typing assistant."</p>
</blockquote>

<h2 id="4-cac-cong-cu-vibe-coding-pho-bien"><strong>4. Popular Vibe Coding Tools (2026)</strong></h2>

<h3>4.1. GitHub Copilot (VS Code)</h3>
<p>Leading AI coding tool from GitHub/Microsoft, deeply integrated into VS Code:</p>
<ul>
<li><strong>Inline Suggestions</strong>: suggests code while typing</li>
<li><strong>Agent Mode</strong>: AI automatically implements features across multiple files</li>
<li><strong>Plan Agent</strong>: plan before coding</li>
<li><strong>Cloud Agent</strong>: run agent on cloud, create PR automatically</li>
<li><strong>Copilot CLI</strong>: agent runs from terminal</li>
<li><strong>Custom Agents & MCPs</strong>: expanding AI capabilities</li>
<li>Supports many models: GPT-5.4, Claude, Gemini 3.1 Pro</li>
</ul>

<h3>4.2. Cursor</h3>
<p>Specialized IDE for AI coding, fork from VS Code. Cursor Composer is an outstanding feature for multi-file editing.</p>

<h3>4.3. Replit Agent</h3>
<p>Cloud IDE platform with integrated AI agent, allowing build and deploy right in the browser.</p>

<h3>4.4. Lovable, Bolt, v0</h3>
<p>"Prompt-to-app" platforms allow the creation of web applications using only text descriptions.</p>

<h2 id="5-khi-nao-nen-vibe-coding"><strong>5. When should Vibe Coding be done?</strong></h2>

<h3>✅ Suitable:</h3>
<ul>
<li><strong>Prototype & MVP</strong>: validate ideas quickly</li>
<li><strong>"Software for One"</strong>: small personal tool (Kevin Roose, NYT)</li>
<li><strong>Weekend side projects</strong>: as Karpathy describes</li>
<li><strong>Learning</strong>: explore new languages/frameworks</li>
<li><strong>Boilerplate code</strong>: CRUD, forms, config files</li>
<li><strong>Rapid iteration</strong>: try many quick approaches</li>
</ul>

<h3>❌ Need to be careful:</h3>
<ul>
<li><strong>Production systems</strong> there are many users</li>
<li><strong>The system handles sensitive data</strong> (healthcare, finance)</li>
<li><strong>Safety-critical code</strong> (medical devices, automotive)</li>
<li><strong>Complex multi-file architectures</strong> not yet documented</li>
<li><strong>Open source maintainability</strong> (paper "Vibe Coding Kills Open Source", Jan 2026)</li>
</ul>

<h2 id="6-han-che-va-rui-ro"><strong>6. Limitations and risks have been noted</strong></h2>

<h3>6.1. Security</h3>
<p>VeraCode (Oct 2025): LLMs are better at generating functional code but security does not improve accordingly. CodeRabbit (December 2025): AI co-authored code available <strong>2.74x more security vulnerabilities</strong> than human-written code.</p>

<h3>6.2. Technical Debt</h3>
<p>GitClear (2025): Code duplication increases 4 times, code churn nearly doubles, refactoring decreases from 25% to less than 10%.</p>

<h3>6.3. Productivity Paradox</h3>
<p>METR (Jul 2025): Experienced open-source developers in practice <strong>19% slower</strong> When using AI tools, self-assessment is 24% faster.</p>

<h3>6.4. Actual Incident</h3>
<ul>
<li><strong>Loveable</strong> (May 2025): 170/1645 apps have vulnerabilities that expose personal information</li>
<li><strong>Replit</strong> (Jul 2025): AI agent deletes production database despite being told not to change anything</li>
<li><strong>Orchids</strong> (Dec 2025): security vulnerability on vibe coding platform</li>
</ul>
