<h2 id="1-fhir-r6-roadmap"><strong>1. FHIR R6 ロードマップ</strong></h2>

<p>FHIR R5 (2023) が現在の標準バージョンです。 R6 は多くの改良を加えて開発されています。</p>

<table>
<thead>
<tr><th>バージョン</th><th>年</th><th>ステータス</th></tr>
</thead>
<tbody>
<tr><td>DSTU1</td><td>2014年</td><td>レガシー</td></tr>
<tr><td>DSTU2</td><td>2015年</td><td>レガシー</td></tr>
<tr><td>STU3</td><td>2017年</td><td>レガシー</td></tr>
<tr><td>R4</td><td>2019年</td><td>標準（最も広く使用されている）</td></tr>
<tr><td>R4B</td><td>2022年</td><td>規範的</td></tr>
<tr><td>R5</td><td>2023年</td><td>規範（現在）</td></tr>
<tr><td>R6</td><td>～2026年</td><td>開発中</td></tr>
</tbody>
</table>

<h3 id="r6-expected"><strong>R6が利用可能になる予定です</strong></h3>

<ul>
<li><strong>サブスクリプションの改善</strong> — トピックベースのサブスクリプションはより完全です</li>
<li><strong>強化された検索</strong> — 複合検索パラメータ、GraphQL の改善</li>
<li><strong>AI/MLの統合</strong> — モデル推論用の標準化されたリソース</li>
<li><strong>デバイス/IoT</strong> — 医療機器とウェアラブルのリソースを改善する</li>
<li><strong>バージョン間の互換性</strong> — ツールはバージョンの移行に適しています</li>
<li><strong>パフォーマンス</strong> — バイナリリソースのストリーミング、ページネーションの改善</li>
</ul>

<h3 id="fhir-ml-pipeline"><strong>FHIR → ML パイプライン</strong></h3>

<pre><code class="language-python"># Extract FHIR data → Train ML model
import requests
import pandas as pd
from sklearn.ensemble import GradientBoostingClassifier

FHIR_BASE = "http://localhost:8080/fhir"

def extract_training_data():
    """Trích xuất dữ liệu từ FHIR Server cho ML training"""
    
    # Bulk export
    patients_bundle = requests.get(
        f"{FHIR_BASE}/Patient?_count=1000"
    ).json()
    
    records = []
    for entry in patients_bundle.get("entry", []):
        patient = entry["resource"]
        pid = patient["id"]
        
        # Lấy Observations
        obs_bundle = requests.get(
            f"{FHIR_BASE}/Observation?subject=Patient/{pid}&code=4548-4"
        ).json()
        
        # Lấy Conditions
        cond_bundle = requests.get(
            f"{FHIR_BASE}/Condition?subject=Patient/{pid}"
        ).json()
        
        record = {
            "patient_id": pid,
            "age": calculate_age(patient.get("birthDate")),
            "gender": 1 if patient.get("gender") == "male" else 0,
            "hba1c": extract_latest_value(obs_bundle),
            "diabetes": has_condition(cond_bundle, "44054006"),
            "hypertension": has_condition(cond_bundle, "38341003"),
        }
        records.append(record)
    
    return pd.DataFrame(records)

# Train model
df = extract_training_data()
X = df[["age", "gender", "hba1c", "diabetes", "hypertension"]]
y = df["cv_event_5yr"]  # label từ follow-up data

model = GradientBoostingClassifier()
model.fit(X, y)
</code></pre>

<h2 id="5-smart-health-cards"><strong>5. SMART ヘルスカード</strong></h2>

</ul>

<h2 id="8-ket-thuc-series"><strong>8. シリーズの終了</strong></h2>

<p>25 のレッスンを通じて、FHIR の旅全体を体験しました。</p>

<ol>
<li><strong>パート 1</strong> — プラットフォーム: HL7 の歴史、FHIR R5 の概要、開発環境</li>
<li><strong>パート 2</strong> — コアリソース: 患者、遭遇、状態、観察、投薬...</li>
<li><strong>パート 3</strong> — RESTful API: CRUD、検索、バンドル、トランザクション</li>
<li><strong>パート 4</strong> — データ型、用語、プロファイル、拡張子、IG</li>
<li><strong>パート 5</strong> — ドキュメント、メッセージング、サブスクリプション、SMART on FHIR、セキュリティ</li>
<li><strong>パート6</strong> — ハンズオン: HAPI FHIR サーバー、クライアント、VN コア IG、EMR/HIS 統合</li>
<li><strong>パート 7</strong> — 制作: パフォーマンス、VN コンテキスト、ケーススタディ、将来</li>
</ol>

<p>FHIR は、世界が医療データを交換する方法を変えています。この知識ベースを使用すると、ベトナムの医療用の相互運用可能なシステムを構築する準備が整います。</p>
