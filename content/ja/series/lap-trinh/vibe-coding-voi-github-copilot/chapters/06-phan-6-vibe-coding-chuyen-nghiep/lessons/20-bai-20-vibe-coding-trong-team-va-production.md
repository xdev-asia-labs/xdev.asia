<h2 id="2-shared-configuration"><strong>2. チームの共有設定</strong></h2>

<h3>2.1.リポジトリレベルの命令</h3>
<pre><code class="language-markdown"><!-- .github/copilot-instructions.md -->
# Team Coding Standards

## Architecture
- Follow Clean Architecture: Controllers → Services → Repositories
- All business logic in Services, never in Controllers
- Use dependency injection via constructor

## Naming Conventions  
- Files: kebab-case (user-service.ts)
- Classes: PascalCase (UserService)
- Functions: camelCase (getUserById)
- Constants: SCREAMING_SNAKE_CASE (MAX_RETRY_COUNT)

## Error Handling
- Use AppError class for all business errors
- HTTP errors only in controllers
- Log errors with structured logging (pino)

## Testing
- Every service method must have unit tests
- Use factories for test data (src/test/factories/)
- Mock external dependencies, not internal modules
</code></pre>

<h2 id="3-cicd-integration"><strong>3. CI/CDの統合</strong></h2>

<h3>3.1.高品質のゲートを備えた完全なパイプライン</h3>
<pre><code class="language-yaml"># .github/workflows/ci.yml
name: CI Pipeline
on: [pull_request]

jobs:
  quality:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: 22 }
      - run: npm ci

      # Type checking
      - run: npx tsc --noEmit

      # Linting
      - run: npx eslint . --max-warnings 0

      # Unit tests
      - run: npm test -- --coverage --forceExit
      - uses: codecov/codecov-action@v4

      # Security scan
      - run: npm audit --audit-level=high

      # Dead code check
      - run: npx knip --no-exit-code

      # Duplication check
      - run: npx jscpd src/ --threshold 3

  integration:
    needs: quality
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_DB: test
          POSTGRES_PASSWORD: test
    steps:
      - uses: actions/checkout@v4
      - run: npm ci
      - run: npx prisma migrate deploy
      - run: npm run test:integration

  # Copilot-powered PR review
  ai-review:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: github/copilot-code-review-action@v1
        with:
          model: gpt-5.4
</code></pre>

<h2 id="6-governance"><strong>6. ガバナンスとコンプライアンス</strong></h2>

<h3>6.1.ライセンス準拠</h3>
<ul>
<li>AIはコピーレフトコード（GPL）に似たコードを生成できる</li>
<li>使用する <strong>副操縦士事業</strong> コード参照フィルターを使用</li>
<li>依存関係ライセンスをスキャンします。 <code>npxライセンスチェッカー</code></li>
</ul>

<h3>6.2.データプライバシー</h3>
<ul>
<li><strong>コパイロット ビジネス/エンタープライズ</strong>: コードはモデルのトレーニングに使用されません</li>
<li>PII、機密情報、顧客データをプロンプトに貼り付けないでください</li>
<li>MCP サーバーはデータ アクセスの監査が必要です</li>
</ul>

<h3>6.3.規制遵守</h3>
<ul>
<li>ヘルスケア (HIPAA): AI による医療データの処理を検討する</li>
<li>財務 (PCI DSS): 支払い処理用の AI コードには追加のレビューが必要です</li>
<li>政府: AI コードの輸出制限を確認する</li>
</ul>
