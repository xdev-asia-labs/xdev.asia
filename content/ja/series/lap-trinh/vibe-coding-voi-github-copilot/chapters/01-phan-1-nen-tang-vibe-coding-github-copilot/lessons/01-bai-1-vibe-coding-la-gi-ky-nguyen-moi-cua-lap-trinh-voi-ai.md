<h2 id="1-vibe-coding-la-gi"><strong>1. バイブコーディングとは何ですか?</strong></h2>

<p><strong>バイブコーディング</strong> (または Vibecoding) は、プログラマーが自然言語で要件を記述し、Large Language Model (LLM) によってソース コードが自動的に生成される、AI を活用したソフトウェア開発手法です。</p>

<p>この用語を作ったのは、 <strong>アンドレイ・カルパシー</strong> — AI 科学者、OpenAI の共同創設者、元 Tesla AI ディレクター — 2025 年 2 月:</p>

<blockquote>
<p>「私が『バイブ コーディング』と呼んでいる新しい種類のコーディングがあります。そこでは、完全にバイブに身を委ね、エクスポネンシャルを受け入れ、コードの存在すら忘れてしまいます。LLM が良すぎるため、それが可能になります。」</p>
<p>— アンドレイ・カルパシー、2025 年 2 月 2 日</p>
</blockquote>

<p>この用語はすぐに非常に人気になり、次のようになりました。</p>
<ul>
<li><strong>メリアム・ウェブスター</strong> 2025 年 3 月に「俗語およびトレンド」としてリストされました。</li>
<li><strong>コリンズ英語辞典</strong> それを選んでください <strong>2025 年の言葉</strong></li>
<li><strong>Y コンビネーター</strong> 2025 年冬期バッチのスタートアップ企業の 25% が、95% が AI によって生成されたコードベースを使用していると報告しました。</li>
</ul>

<h2 id="2-cach-vibe-coding-hoat-dong"><strong>2. Vibe コーディングの仕組み</strong></h2>

<p>従来のプロセスの代わりに <em>考える → コードを書く → テスト → デバッグ</em>、プロセスに従って Vibe コーディング:</p>

<pre><code>┌─────────────────────────────────────────────────┐
│            VIBE CODING WORKFLOW                  │
├─────────────────────────────────────────────────┤
│                                                  │
│  1. 💭 Describe (Mô tả yêu cầu bằng tiếng Việt/│
│        English cho AI)                           │
│           ↓                                      │
│  2. 🤖 Generate (AI sinh code tự động)           │
│           ↓                                      │
│  3. 👀 Review/Accept (Xem kết quả, accept/reject)│
│           ↓                                      │
│  4. 🔄 Iterate (Nếu chưa đúng, mô tả lại)      │
│           ↓                                      │
│  5. ✅ Ship (Deploy sản phẩm)                    │
│                                                  │
└─────────────────────────────────────────────────┘
</code></pre>

<p>カルパシーは自身の経験を次のように説明しています。</p>
<ul>
<li>AIと音声で会話（SuperWhisper）</li>
<li>差分を読まずに常に「すべて受け入れる」を押してください。</li>
<li>エラー メッセージをコメントなしでコピーアンドペーストします。</li>
<li>コードは開発者の理解能力を超えて進化する</li>
</ul>

<h2 id="4-cac-cong-cu-vibe-coding-pho-bien"><strong>4. 人気の Vibe コーディング ツール (2026)</strong></h2>

<h3>4.1. GitHub コパイロット (VS コード)</h3>
<p>GitHub/Microsoft の最先端の AI コーディング ツールで、VS Code に深く統合されています。</p>
<ul>
<li><strong>インライン提案</strong>: 入力中にコードを提案します</li>
<li><strong>エージェントモード</strong>: AI が複数のファイルにわたって機能を自動的に実装します</li>
<li><strong>プランエージェント</strong>: コーディングの前に計画を立てる</li>
<li><strong>クラウドエージェント</strong>: クラウド上でエージェントを実行し、PR を自動的に作成します</li>
<li><strong>コパイロット CLI</strong>: エージェントは端末から実行されます</li>
<li><strong>カスタムエージェントとMCP</strong>: AI 機能の拡張</li>
<li>多くのモデルをサポート: GPT-5.4、Claude、Gemini 3.1 Pro</li>
</ul>

<h3>4.2.カーソル</h3>
<p>AI コーディングに特化した IDE、VS Code からフォーク。 Cursor Composer は、複数ファイルの編集に優れた機能です。</p>

<h3>4.3.エージェントの複製</h3>
<p>統合された AI エージェントを備えたクラウド IDE プラットフォームにより、ブラウザーで直接構築およびデプロイが可能になります。</p>

<h3>4.4.愛すべきボルト、v0</h3>
<p>「Prompt-to-app」プラットフォームでは、テキストの説明のみを使用して Web アプリケーションを作成できます。</p>

<h2 id="5-khi-nao-nen-vibe-coding"><strong>5. Vibe コーディングはいつ行うべきですか?</strong></h2>

<h3>✅ 適したもの:</h3>
<ul>
<li><strong>プロトタイプと MVP</strong>: アイデアを素早く検証する</li>
<li><strong>「一人用ソフトウェア」</strong>: 小さな個人用ツール (ニューヨークタイムズ、ケビン・ルース)</li>
<li><strong>週末のサイドプロジェクト</strong>：カルパシーの説明によると</li>
<li><strong>学習</strong>: 新しい言語/フレームワークを探索する</li>
<li><strong>定型コード</strong>: CRUD、フォーム、構成ファイル</li>
<li><strong>迅速な反復</strong>: 簡単なアプローチをたくさん試してください</li>
</ul>

<h3>❌ 注意が必要な点:</h3>
<ul>
<li><strong>生産体制</strong> 多くのユーザーがいます</li>
<li><strong>システムは機密データを処理します</strong> (ヘルスケア、金融)</li>
<li><strong>セーフティクリティカルなコード</strong> (医療機器、自動車)</li>
<li><strong>複雑なマルチファイルアーキテクチャ</strong> まだ文書化されていない</li>
<li><strong>オープンソースの保守性</strong> (論文「Vibecoding Kills Open Source」、2026 年 1 月)</li>
</ul>

<h2 id="6-han-che-va-rui-ro"><strong>6. 制限とリスクが指摘されている</strong></h2>

<h3>6.1.セキュリティ</h3>
<p>VeraCode (2025 年 10 月): LLM は関数コードの生成には優れていますが、それに応じてセキュリティは向上しません。 CodeRabbit (2025 年 12 月): AI 共同作成コードが利用可能 <strong>2.74 倍のセキュリティ脆弱性</strong> 人間が書いたコードよりも。</p>

<h3>6.2.技術的負債</h3>
<p>GitClear (2025): コードの重複は 4 倍に増加し、コード チャーンはほぼ 2 倍になり、リファクタリングは 25% から 10% 未満に減少しました。</p>

<h3>6.3.生産性のパラドックス</h3>
<p>METR (2025 年 7 月): 経験豊富なオープンソース開発者が実践 <strong>19% 遅くなる</strong> AI ツールを使用すると、自己評価が 24% 高速になります。</p>

<h3>6.4.実際の事件</h3>
<ul>
<li><strong>愛らしい</strong> (2025 年 5 月): 170/1645 アプリに個人情報を漏らす脆弱性がある</li>
<li><strong>Replit</strong> (2025 年 7 月): 何も変更しないように指示されているにもかかわらず、AI エージェントが実稼働データベースを削除</li>
<li><strong>Orchids</strong> (2025 年 12 月): バイブコーディングプラットフォームのセキュリティ脆弱性</li>
</ul>
