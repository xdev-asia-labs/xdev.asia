技術的に優秀だがステークホルダーを説得できないBA = プロジェクトの失敗。すべてに長けているがインタビューができないBA = 証明する機会がない。AIシミュレーションは、フィードバック付きで、費用なし、回数制限なしで練習する方法です。

---

## 1. AIによるステークホルダーシミュレーション

### 1.1 なぜシミュレーションが効果的なのか？

難しいステークホルダーには繰り返されるパターンがあります：
- **懐疑論者：** 「AIは本当に必要ですか？手動でも十分では？」
- **過剰要求者：** 「この機能も追加してください — そんなに時間かかりませんよね？」
- **リスク回避者：** 「AIが間違えたらどうするの？私はサインオフしません。」
- **予算管理者：** 「このROIは確実ですか？計算根拠は？」
- **テクニカルチャンピオン：** AIにすべてをやらせたい、ユーザーニーズを聞こうとしない

AIシミュレーションで練習する → パターンを認識する → 実際に遭遇したときにうまく対処できる。

### 1.2 ステークホルダーシミュレーション用プロンプトテンプレート

**セットアッププロンプト：**
```
あなたは[ステークホルダー名]、[X業界の会社]の[役職]です。
あなたの特徴は以下の通りです：
- [特徴1：例：「コミットする前に常にROIと数字を尋ねる」]
- [特徴2：例：「前のAIプロジェクトが失敗したのでAIに懐疑的」]
- [特徴3：例：「予算権限はあるが技術に詳しくない」]

私（BA）はAIフィーチャーの提案をあなたに発表します。
キャラクターに沿って反応し、難しい質問をし、必要なときはプッシュバックしてください。
開始の一言：「OK、20分あります。このフィーチャーが何か教えてください。」
```

**シナリオ例：**
```
シナリオ：AIチャットボットの5万ドルの予算承認をCFOに説得する必要がある。
CFOは以前に「ROIが明確でない」として2件のAI提案を却下している。
私は15分と5枚のスライドがある。
```

### 1.3 シミュレーションセッションの種類

**セッションタイプ1：ディスカバリーミーティング**
```
シナリオ：新しいステークホルダー、要件を抽出するための最初のミーティング。
目標：会話の開き方、オープンエンドの質問の仕方、
       積極的傾聴、誘導質問を避ける方法を学ぶ。
```

**セッションタイプ2：プッシュバックミーティング**
```
シナリオ：ステークホルダーがあなたの要件優先順位付けに同意しない。
目標：反論の処理方法、冷静さを保つ方法、地盤を失わずに交渉する方法を学ぶ。
```

**セッションタイプ3：エグゼクティブプレゼンテーション**
```
シナリオ：ゴーライブの承認を得るためにディレクターへの10分間のスロット。
目標：明確さ、自信、厳しいQ&Aへの対応。
```

## 2. AIによる模擬面接

### 2.1 BAのインタビュータイプとシミュレーション

**タイプ1：行動面接（STAR形式）**

```
AIへのプロンプト：
「AIを専門とするシニアビジネスアナリストのポジションの面接をしてください。
5つの行動質問をしてください。各質問の後：
1. 質問を尋ねる
2. 私が答えたら、具体的なフィードバックを提供する：
   - 回答の強み
   - 弱点または不足している点
   - STAR形式での改善提案」
```

**タイプ2：ケーススタディ面接**

```
AIへのプロンプト：
「BAのケーススタディを提示してください：
 フィンテック企業がAIで信用審査を自動化したい。
 難しい面接官のように質問し、
 問題の構造化、要件の特定、
 曖昧さへの対処方法を評価してください。」
```

**タイプ3：技術的深度チェック**

```
AIへのプロンプト：
「AIについてのBA知識をテストしてください。順番に以下について質問してください：
1. 精度と再現率の違い — それぞれいつ使うか？
2. BAはAIモデルのテストセットをどのように設計するか？
3. なぜ人間が介在するループが必要なのか？
4. GDPR第22条はAIフィーチャーの要件にどのような影響を与えるか？
各回答の後、1〜5のスコアを付けて理由を説明してください。」
```

## 3. フィードバックループ：シミュレーションを真に価値あるものにする

### 3.1 各シミュレーションセッション後

```
フィードバックを要求するプロンプト：
「今からキャラクターを離れてください。
先ほどのシミュレーションでの私のパフォーマンスを評価してください：
1. 最も優れた回答や主張はどれでしたか？なぜですか？
2. どこが弱点だったり、欠けていましたか？
3. 私が準備できていなかったステークホルダーの質問はありましたか？
4. 次回改善すべき3つのこと？」
```

### 3.2 進捗追跡テンプレート

```markdown

| 日付 | タイプ | シナリオ | スコア（1〜5） | 主な弱点 | 次回改善した？ |

| YYYY-MM-DD | ステークホルダー | CFO予算承認 | 3/5 | 具体的なROI数値なし | ☐ |
| YYYY-MM-DD | 模擬面接 | 行動 - 対立処理 | 4/5 | STAR構造が弱い | ☐ |
```

## 4. 上級：難易度を上げるAIロールプレイ

### 段階的に難易度を上げる

**レベル1：** AIが協力的で支援的なステークホルダーを演じる  
**レベル2：** AIが懐疑的だが合理的なステークホルダーを演じる  
**レベル3：** AIが独自のアジェンダを持つ敵対的なステークホルダーを演じる  
**レベル4：** AIが複数のステークホルダーを同時に演じる（利益の対立）

```
レベル4プロンプト：
「同じミーティングで2人のステークホルダーを演じてください：
- A：CTO、AIフィーチャーをできるだけ早く欲しい、ビジネスケースを気にしない
- B：CFO、明確なROIを要求、AIに懐疑的

2人はダイナミックな緊張関係にある。私は両方を説得しなければならない。
ミーティングを始めてください。」
```

## まとめ

AIシミュレーションは実際の経験の代替にはなりません — しかし、準備速度を**加速**します。AIによる10回のシミュレーションセッション＋1回の実際のステークホルダーミーティング = 実際のミーティング1回だけよりはるかに良い結果。

難しい質問に**飽きる**まで練習してください — それが本番の準備ができているサインです。
