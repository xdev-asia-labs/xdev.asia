<h2 id="1-vibe-coding-la-gi"><strong>1. 什麼是 Vibe 編碼？</strong></h2>

<p><strong>氛圍編碼</strong> （或稱 Vibecoding）是一種人工智慧驅動的軟體開發方法，程式設計師用自然語言描述需求，並讓大型語言模型（LLM）自動產生原始程式碼。</p>

<p>這個術語是由 <strong>安德烈·卡帕蒂</strong> — AI 科學家、OpenAI 共同創辦人、特斯拉前 AI 總監 — 2025 年 2 月：</p>

<blockquote>
<p>“有一種新的編碼方式，我稱之為‘氛圍編碼’，你可以完全屈服於氛圍，擁抱指數，甚至忘記代碼的存在。這是可能的，因為法學碩士變得太好了。”</p>
<p>— 安德烈·卡帕蒂 (Andrej Karpathy)，2025 年 2 月 2 日</p>
</blockquote>

<p>這個詞很快就流行起來：</p>
<ul>
<li><strong>韋氏字典</strong> 2025 年 3 月將其列為“俚語和趨勢”</li>
<li><strong>柯林斯英語字典</strong> 選擇它 <strong>2025 年年度詞彙</strong></li>
<li><strong>Y組合器</strong> 據報道，2025 年冬季批次中 25% 的新創公司擁有 95% 由人工智慧產生的程式碼庫</li>
</ul>

<h2 id="2-cach-vibe-coding-hoat-dong"><strong>2. Vibe 編碼的工作原理</strong></h2>

<p>代替傳統工藝 <em>思考→寫程式→測試→調試</em>，Vibe Coding依照流程：</p>

<pre><code>┌─────────────────────────────────────────────────┐
│            VIBE CODING WORKFLOW                  │
├─────────────────────────────────────────────────┤
│                                                  │
│  1. 💭 Describe (Mô tả yêu cầu bằng tiếng Việt/│
│        English cho AI)                           │
│           ↓                                      │
│  2. 🤖 Generate (AI sinh code tự động)           │
│           ↓                                      │
│  3. 👀 Review/Accept (Xem kết quả, accept/reject)│
│           ↓                                      │
│  4. 🔄 Iterate (Nếu chưa đúng, mô tả lại)      │
│           ↓                                      │
│  5. ✅ Ship (Deploy sản phẩm)                    │
│                                                  │
└─────────────────────────────────────────────────┘
</code></pre>

<p>卡帕蒂描述了他的經歷：</p>
<ul>
<li>透過語音與 AI 交談 (SuperWhisper)</li>
<li>始終按“全部接受”而不閱讀差異</li>
<li>複製貼上錯誤訊息，沒有任何進一步的註釋</li>
<li>程式碼的演變超出了開發人員的理解能力</li>
</ul>

</blockquote>

<h2 id="4-cac-cong-cu-vibe-coding-pho-bien"><strong>4.流行的 Vibe 編碼工具 (2026)</strong></h2>

<h3>4.1. GitHub Copilot（VS 程式碼）</h3>
<p>來自 GitHub/Microsoft 的領先 AI 編碼工具，深度整合到 VS Code 中：</p>
<ul>
<li><strong>內嵌建議</strong>：輸入時建議代碼</li>
<li><strong>代理模式</strong>：AI自動跨多個檔案實作功能</li>
<li><strong>計劃代理</strong>：編碼前計劃</li>
<li><strong>雲端代理</strong>：雲端運行agent，自動建立PR</li>
<li><strong>副駕駛 CLI</strong>：代理從終端運行</li>
<li><strong>客製化代理商和 MCP</strong>：擴展人工智慧能力</li>
<li>支援多種型號：GPT-5.4、Claude、Gemini 3.1 Pro</li>
</ul>

<h3>4.2.遊標</h3>
<p>用於 AI 編碼的專用 IDE，源自 VS Code。 Cursor Composer 是多文件編輯的一項出色功能。</p>

<h3>4.3.複製代理</h3>
<p>具有整合 AI 代理的雲端 IDE 平台，允許直接在瀏覽器中建置和部署。</p>

<h3>4.4.可愛的博爾特 v0</h3>
<p>「Prompt-to-app」平台允許僅使用文字描述建立 Web 應用程式。</p>

<h2 id="5-khi-nao-nen-vibe-coding"><strong>5. Vibe Coding應該什麼時候進行？</strong></h2>

<h3>✅ 適合：</h3>
<ul>
<li><strong>原型和 MVP</strong>：快速驗證想法</li>
<li><strong>“一個人的軟體”</strong>：小型個人工具（Kevin Roose，紐約時報）</li>
<li><strong>週末業餘項目</strong>：正如卡帕蒂所描述的</li>
<li><strong>學習</strong>：探索新的語言/框架</li>
<li><strong>樣板程式碼</strong>：CRUD、表單、設定檔</li>
<li><strong>快速迭代</strong>：嘗試許多快速方法</li>
</ul>

<h3>❌ 需要注意的是：</h3>
<ul>
<li><strong>生產系統</strong> 有很多用戶</li>
<li><strong>系統處理敏感數據</strong> （醫療保健、金融）</li>
<li><strong>安全關鍵程式碼</strong> （醫療器材、汽車）</li>
<li><strong>複雜的多文件架構</strong> 尚未記錄</li>
<li><strong>開源可維護性</strong> （論文“Vibe Coding Kills Open Source”，2026 年 1 月）</li>
</ul>

<h2 id="6-han-che-va-rui-ro"><strong>6. 限制和風險已被注意到</strong></h2>

<h3>6.1.安全性</h3>
<p>VeraCode（2025 年 10 月）：法學碩士更擅長產生功能代碼，但安全性並沒有相應提高。 CodeRabbit（2025 年 12 月）：AI 共同編寫的程式碼可用 <strong>安全漏洞增加 2.74 倍</strong> 比人類編寫的程式碼。</p>

<h3>6.2.技術債</h3>
<p>GitClear (2025)：程式碼重複增加 4 倍，程式碼改變幾乎翻倍，重構從 25% 減少到不到 10%。</p>

<h3>6.3.生產力悖論</h3>
<p>METR（2025 年 7 月）：經驗豐富的開源開發人員的實踐 <strong>慢 19%</strong> 使用 AI 工具時，自我評估速度提高了 24%。</p>

<h3>6.4.實際事件</h3>
<ul>
<li><strong>可愛的</strong> （2025 年 5 月）：170/1645 個應用程式存在洩露個人資訊的漏洞</li>
<li><strong>雷普利特</strong> （2025 年 7 月）：AI 代理刪除了生產資料庫，儘管被告知不要更改任何內容</li>
<li><strong>蘭花</strong> （2025 年 12 月）：vibe 編碼平台上的安全漏洞</li>
</ul>

<h2 id="7-tai-sao-hoc-series-nay"><strong>7. 為什麼要學本系列？</strong></h2>
