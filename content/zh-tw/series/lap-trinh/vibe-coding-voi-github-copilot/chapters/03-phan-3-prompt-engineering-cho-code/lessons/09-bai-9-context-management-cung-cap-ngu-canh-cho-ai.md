<h2 id="1-context-la-gi"><strong>1. 什麼是上下文？為什麼它很重要？</strong></h2>

<p>AI模型只能「看到」裡面的東西 <strong>上下文視窗</strong> — 每個請求處理的文字量。如果你沒有提供足夠的上下文，AI就會“猜測”，而且經常猜錯。</p>

<pre><code class="language-text">Context window: [System prompt + Custom instructions + Your prompt + Attached files + History]
                ↑ tất cả phải fit trong giới hạn tokens của model
</code></pre>

<p><strong>良好的上下文=良好的程式碼。</strong> 這是有效 Vibe 編碼的第一原則。</p>

<h2 id="2-cac-loai-context"><strong>2. Copilot 中的上下文類型</strong></h2>

<h3>2.1.隱式上下文（自動）</h3>
<p>Copilot 自動收集：</p>
<ul>
<li><strong>文件處於活動狀態</strong> 在編輯器中</li>
<li><strong>選擇</strong> 目前（如果有）</li>
<li><strong>打開選項卡</strong></li>
<li><strong>工作區結構</strong> （文件/資料夾樹）</li>
<li><strong>診斷</strong> （錯誤、警告）</li>
<li><strong>端子輸出</strong> 最近的</li>
</ul>

<h3>2.2.顯式上下文（您提供）</h3>
<p>您主動附加：</p>
<ul>
<li><strong>#文件</strong>: 附上具體文件</li>
<li><strong>#選擇</strong>: 正在選擇的代碼</li>
<li><strong>#編輯器</strong>：編輯器中的整個可見區域</li>
<li><strong>#程式碼庫</strong>：在工作區中搜尋語義</li>
<li><strong>#terminalLastCommand</strong>：剛剛運行的命令的輸出</li>
<li><strong>拖放</strong> 文件進入聊天室</li>
<li><strong>圖片</strong>：附上螢幕截圖（2026 年 3 月專題）</li>
</ul>

<h2 id="3-su-dung-context-variables"><strong>3.使用上下文變數</strong></h2>

<h3>3.1. #file — 附加特定文件</h3>
<pre><code class="language-text">// Trong Chat view:
Look at #file:src/models/User.ts and #file:src/services/AuthService.ts
and create a password reset flow following the same patterns.
</code></pre>

<p>打字時 <code>#文件：</code>，VS Code 顯示檔案選擇器供您選擇。</p>

<h3>3.2. #selection — 程式碼正在選擇</h3>
<pre><code class="language-text">// Select đoạn code trong editor, rồi trong Chat:
Explain #selection and suggest how to optimize it for performance.
</code></pre>

<h3>3.3. #codebase — 在工作區中搜尋語意</h3>
<pre><code class="language-text">// Copilot tự tìm files liên quan trong codebase:
#codebase How is authentication implemented in this project?
Show me the auth middleware and token verification logic.
</code></pre>

<h3>3.4. #terminalLastCommand — 輸出終端</h3>
<pre><code class="language-text">// Sau khi chạy test fail trong terminal:
#terminalLastCommand Fix the failing tests. Analyze the error output
and make the necessary code changes.
</code></pre>

<h2 id="4-workspace-indexing"><strong>4. 工作區索引</strong></h2>

<p>VS Code 自動索引工作區，以便 Copilot 了解程式碼庫：</p>

<ul>
<li><strong>語意索引</strong>：理解程式碼的意義（函數、類別、關係）</li>
<li><strong>符號索引</strong>：函數名、變數、型別</li>
<li><strong>文件結構</strong>: 了解專案結構</li>
</ul>

<p>Agent運行時可以使用諸如 <code>搜尋。搜尋</code>, <code>列表目錄</code>, <code>讀</code> 主動探索程式碼庫——而不僅僅是基於您提供的上下文。</p>

<h2 id="5-chien-luoc-context"><strong>5. 有效的情境策略</strong></h2>

<h3>5.1.在出現提示之前開啟正確的文件</h3>
<pre><code class="language-text">// Trước khi viết prompt "Create UserService":
// Mở các files liên quan trong editor tabs:
Tab 1: models/User.ts        → Copilot biết User schema
Tab 2: services/AuthService.ts → Copilot biết coding pattern
Tab 3: routes/authRoutes.ts   → Copilot biết route conventions
Tab 4: tests/auth.test.ts     → Copilot biết testing pattern
</code></pre>

<h3>5.2.參考現有模式</h3>
<pre><code class="language-text">// Thay vì mô tả pattern từ đầu:
Create a ProductService following the exact same pattern as
#file:src/services/UserService.ts — same error handling,
same repository pattern, same validation approach.
</code></pre>

<h3>5.3.提供明確的約束</h3>
<pre><code class="language-text">// Context about project conventions:
Our project uses:
- Prisma for database (schema at #file:prisma/schema.prisma)
- Zod for validation (see pattern in #file:src/schemas/userSchema.ts)
- Custom AppError class for errors (#file:src/utils/AppError.ts)

Now create the OrderService with CRUD operations.
</code></pre>

<h2 id="6-new-session-vs-continue"><strong>6. 什麼時候是新會話與連續會話？</strong></h2>

<table>
<thead>
<tr>
<th>情況</th>
<th>行動</th>
<th>原因</th>
</tr>
</thead>
<tbody>
<tr>
<td>繼續使用相同的功能</td>
<td><strong>繼續</strong></td>
<td>保留上下文+歷史</td>
</tr>
<tr>
<td>全新功能</td>
<td><strong>新會話</strong></td>
<td>環境乾淨，沒有噪音</td>
</tr>
<tr>
<td>上下文太長</td>
<td><strong>新會話</strong> + 總結</td>
<td>避免上下文溢出</td>
</tr>
<tr>
<td>代理人走向錯誤的方向</td>
<td><strong>新會話</strong> +改寫</td>
<td>重置歷史錯誤</td>
</tr>
<tr>
<td>除錯相關問題</td>
<td><strong>繼續</strong></td>
<td>人工智慧首先需要知道如何修復它</td>
</tr>
</tbody>
</table>

<h3>提示：建立新會話時的摘要</h3>
<pre><code class="language-text">// Khi context quá dài, tạo new session với summary:
Context from previous session:
- We're building a task management app with Next.js 15 + Prisma
- Backend API is done (auth, CRUD for tasks, projects)
- Database schema is finalized
- Currently working on frontend

Continue from: Create the Task list page component with
filtering and sorting capabilities.
</code></pre>

<h2 id="7-images-as-context"><strong>7. 圖像作為背景（2026 年專題）</strong></h2>

<p>從 2026 年 3 月起，您可以 <strong>附加影像</strong> 轉至代理會話：</p>

<ul>
<li>UI設計截圖→“建立這個UI”</li>
<li>錯誤的螢幕截圖→“修復此錯誤”</li>
<li>圖表/線框→“實現此架構”</li>
<li>Figma 匯出→“將此設計轉換為 React 元件”</li>
</ul>

<h2 id="8-anti-patterns"><strong>8. 應避免反模式</strong></h2>

<ul>
<li><strong>轉儲整個程式碼庫</strong>：太多背景 = 噪音，AI 很困惑</li>
<li><strong>未提供上下文</strong>：人工智慧必須猜測一切</li>
<li><strong>附加文件不相關</strong>: 浪費上下文視窗</li>
<li><strong>繼續會話時間太長</strong>：上下文退化，AI“忘記”第一個訊息</li>
<li><strong>複製貼上錯誤，沒有格式化</strong>：為人工智慧帶來混亂</li>
</ul>

<h2 id="9-tong-ket"><strong>9. 總結</strong></h2>

<p>情境管理是一項技能 <strong>決定 Vibe 編碼的質量</strong>：</p>

<ul>
<li>✅ 開啟正確的檔案 → 良好的隱式上下文</li>
<li>✅ 正確使用#file、#selection、#codebase → 明確上下文</li>
<li>✅ 參考現有模式 → 一致的輸出</li>
<li>✅ 當上下文臃腫時新會話 → 避免降級</li>
<li>✅ 為 UI 任務附加圖像 → 視覺上下文</li>
</ul>
