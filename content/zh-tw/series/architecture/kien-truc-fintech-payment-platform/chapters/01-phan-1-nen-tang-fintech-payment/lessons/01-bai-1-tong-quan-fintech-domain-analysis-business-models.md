## 簡介

FinTech（金融科技）徹底改變了我們與金融服務互動的方式。從咖啡店二維碼支付到5分鐘在線借錢——所有這些都是由幕後金融科技系統操作的。

在第一課中，我們將全面分析金融科技領域，並了解主要垂直領域、商業模式和監管環境—尤其是在越南市場。

---

## 1.什麼是金融科技？

### 1.1 定義

**金融科技**是**金融**和**技術**的結合—利用科技來改善、自動化和顛覆傳統金融服務。

```
FinTech = Financial Services + Modern Technology
        = Traditional Banking Problems + Software Solutions
```

### 1.2 金融科技的演變

|相|期間 |特點|
|------------|---------|------------|
| **金融科技1.0** | 1866-1967 |電報、跨大西洋電纜、電傳|
| **金融科技2.0** | 1967-2008 | ATM、SWIFT、網路銀行 |
| **金融科技3.0** | 2008-2020 |行動支付、P2P 借貸、新銀行 |
| **金融科技4.0** | 2020 年至今 |嵌入式金融、DeFi、人工智慧驅動、超級應用程式 |

### 1.3 為什麼金融科技對軟體工程師很重要？

- **龐大的市場**：到 2026 年，全球金融科技市場將達到 $340B
- **越南市場**：50+百萬電子錢包用戶，80%的人口擁有智慧型手機
- **高薪**：金融科技工程師擁有市場上最高的薪水
- **技術挑戰**：高吞吐量、一致性、安全性、合規性

## 2. 金融科技的主要領域

### 2.1 付款 — 付款

最大和最受歡迎的垂直行業：

```
┌─────────────────────────────────────────────┐
│              PAYMENTS ECOSYSTEM              │
├─────────────┬──────────────┬────────────────┤
│   Online    │   In-store   │    P2P/P2M     │
│  Payments   │  (POS/QR)    │   Transfer     │
├─────────────┼──────────────┼────────────────┤
│ Payment     │ Merchant     │ Cross-border   │
│ Gateway     │ Acquiring    │ Remittance     │
├─────────────┼──────────────┼────────────────┤
│ E-wallet    │ Buy Now      │ Subscription   │
│             │ Pay Later    │ Billing        │
└─────────────┴──────────────┴────────────────┘
```

**越南例子**：VNPay、MoMo、ZaloPay、ShopeePay、VietQR

### 2.2 借貸－借貸

```
Consumer Lending          Business Lending
├── Personal loans        ├── SME loans
├── Buy Now Pay Later     ├── Invoice financing
├── Microloans            ├── Supply chain finance
└── Student loans         └── Revenue-based financing
```

**範例**：MoMo（消費貸款）、Tima、Fiin Credit

### 2.3 數位銀行－數位銀行

- **Neobanks**：完全數位銀行（無分行）
- **挑戰者銀行**：傳統銀行+數位轉型
- **銀行即服務 (BaaS)**：API 優先的銀行基礎設施

**越南的例子**：Timo、TNEX、VPBank 的 Cake

### 2.4 InsurTech — 技術保險

- 按需保險、參數保險
- 人工智慧索賠自動化
- 基於遠端資訊處理的定價

### 2.5 財富科技與投資

- 機器人顧問、部分投資
- 社交交易、跟單交易
- 加密貨幣交易所和 DeFi

## 5. 監管環境

### 5.1 越南法規

|法規|內容 |適用於 |
|----------|---------|-------------|
| **第 101/2012 號法令** |非現金支付 |支付中介|
| **第 39/2014 號通知** |提供支付中介服務的活動|電子錢包、支付網關 |
| **第 35/2023 號法令** |非現金支付修正案|所有支付服務 |
| **2023 年電子交易法** |數位簽章、電子合約|數位合約|
| **金融科技沙盒** |金融科技監理沙箱|金融科技新品 |

### 5.2 國際法規

- **PCI-DSS**：支付卡產業資料安全標準
- **PSD2/PSD3**：支付服務指令（歐盟）
- **SOX**：薩班斯-奧克斯利法案（財務報告）
- **GDPR/PDPA**：資料保護法規
- **巴塞爾協議 III**：銀行資本要求

## 7. 金融科技的技術挑戰

### 7.1 主要挑戰

|挑戰|描述 |解決方案 |
|------------|--------|------------|
| **一致性** |不虧錢 | ACID、Saga、冪等性 |
| **可用性** | 99.99%+ 正常運作時間 |多區域、故障轉移 |
| **吞吐量** |百萬TPS峰值|水平縮放 |
| **延遲** | < 100ms response | Caching, async processing |
| **Security** | PCI-DSS, encryption | HSM, tokenization, WAF |
| **Compliance** | Regulatory reporting | Audit trail, immutable logs |
| **Reconciliation** | Exact matching | Double-entry ledger |

### 7.2 FinTech System Characteristics

```
Financial System Requirements:
├── Correctness > 業績（錢不能丟）
├── 可審計（每一筆交易都必須可追溯）
├── 冪等性（重複請求必須安全）
├── 一致性（餘額必須永遠正確）
├── 安全（縱深防禦）
└── 合規（必須符合監管要求）
```
