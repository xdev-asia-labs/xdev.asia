BA 日益使用 AI——總結文件、起草驗收標準、審查需求、撰寫測試情況。但如果提示寫得不仔細，AI 輸出將是通俗、脈絡不適當或缺少所需格式。

本指南不教 AI 工程。它教 BA **為日常工作寫足夠好的提示**。

---

## 1. RPCF 框架

有效提示的四個要素：

| 要素 | 意義 | 例子 |
|------|------|------|
| **R**ole | AI 扮演什麼角色？ | 「你是 10 年 Fintech 經驗的資深 BA」 |
| **P**urpose | 提示的具體目標 | 「檢查下列驗收標準」 |
| **C**ontext | 必要的背景 | 「項目是小零售客戶的銀行行動應用」 |
| **F**ormat | 期望的輸出格式 | 「以要點形式返回，最多 10 項」 |

**沒有 RPCF 的提示：**
> 「檢查我的驗收標準。」

**有 RPCF 的提示：**
> 「你是有敏捷經驗的資深 BA。檢查銀行行動應用內轉帳功能的以下驗收標準。目標：尋找遺漏的邊界情況和邏輯矛盾。返回格式：(1) 遺漏的邊界情況、(2) 邏輯矛盾、(3) 建議——每部分最多 5 項。
> [PASTE AC]」

## 2. BA 常用提示模式

### 模式 1：審查與評論

```
你是 [角色]。針對 [具體基準] 審查 [文件/成果物]。
脈絡：[項目/領域描述]。
返回：[輸出格式]。
---
[PASTE CONTENT]
```

### 模式 2：生成初稿

```
為 [對象] 建立 [文件類型]，條件 [條件/限制]。
脈絡：[項目脈絡]。
格式：[項目數、標題等級、語言]。
新增：[特殊元素如有，例如「AI 功能的邊界情況」]。
```

### 模式 3：轉換格式

```
將 [輸入格式] 轉換為以下 [輸出格式]。
保留不變：[什麼不該改變]。
新增：[應該添加什麼（如果缺少）]。
---
[PASTE CONTENT]
```

### 模式 4：問答・澄清模擬

```
你是 [利益關係人角色：PM/CTO/終端使用者/合規]。
我將呈現需求。對這些需求提出 [此角色] 會問的 5 個最難的問題。
---
[PASTE REQUIREMENTS]
```

## 3. BA 日常工作提示

### 總結長文件

```
從 BA 的角度總結以下文件——我需要什麼來撰寫需求：
- 主要業務目標
- 利益關係人和角色
- 重要限制（時間、預算、合規）
- 需要澄清的模糊點
最多 300 字。
---
[PASTE DOCUMENT]
```

### 建立面試問題

```
我要面試 [利益關係人角色] 關於 [項目描述]。
以下結構建立 10 個發現問題：
- 關於現況流程的 3 個
- 關於痛點的 3 個
- 關於期望/成功標準的 2 個
- 關於限制的 2 個

避免是/否問題。優先開放式問題。
```

### 審查 BRD / SRS

```
審查以下 BRD 並評估：
1. 哪些需求模糊或無法衡量？
2. 缺少的非功能需求？
3. 需求間的矛盾？
4. 未清楚陳述的假設？

以表格返回：| 需求 ID | 問題 | 建議修正 |
---
[PASTE BRD SECTION]
```

### 從 AC 撰寫測試情景

```
從以下驗收標準，以 Given/When/Then 格式撰寫測試情景：
- 3 個成功路徑情景
- 3 個錯誤/邊界情況情景
- 2 個效能/負載情景

脈絡：[功能描述和環境]
---
[PASTE ACCEPTANCE CRITERIA]
```

## 4. 進階技巧：鏈式提示

針對複雜任務，別強行把所有東西放進 1 個提示。改為連鎖多個步驟：

**例子：面試轉錄 → 使用者故事**

```
步驟 1：「將轉錄總結為優先順序排序的問題點清單」
步驟 2：「從這些問題點，按業務領域建立史詩」  
步驟 3：「將史詩 [X] 展開為 3-5 個遵循 INVEST 的使用者故事」
步驟 4：「為故事 [Y]，以 Given/When/Then 格式撰寫驗收標準」
```

益處：BA 在傳給下一步前審查每步輸出。減少複合誤差。

## 5. 產品內 AI 功能的提示

BA 當規格化 AI 功能時，定義產品的系統提示：

### 系統提示範本

```markdown
## 系統提示範本 [功能名稱]

**角色分配：**
你是 [AI 在產品中的角色，例如「銀行應用的客服助理」]。

**範圍：**
只回答關於 [特定領域] 的問題。
不要回答關於 [超出範圍主題] 的問題。

**語調和格式：**
- 語言：繁體中文、禮貌、專業
- 回應長度：最多 [X] 句
- 格式：[純文本 / Markdown / 要點清單]

**升級：**
當問題涉及 [觸發關鍵字清單] 時，說：
「我需要將你連接給專家。請稍候。」

**永不：**
- 對 [機密主題] 給建議
- 通過聊天確認帳戶資訊
- 承諾政策外的東西
```

## 6. 評估提示品質

好提示產生：
- 一致輸出（相同輸入 → 相同一般輸出）
- 脈絡正確答案
- 所要求格式的輸出
- 最小幻覺

在本機使用前測試提示：
- 相同提示、不同輸入 → 一致風格？
- 輸出符合業務脈絡？
- 格式總是對的？
- 有明顯幻覺嗎？

---

## 總結

提示設計是 BA 技能。不是 AI 工程，但與 AI 的專業溝通。掌握 RPCF，系統地測試，反覆。設計妥當的提示節省數小時重做；模糊提示浪費時間重做。

開始於：定義角色 + 目的 + 脈絡 + 格式。測試。反覆。保留並重用最佳提示作為公司範本。
